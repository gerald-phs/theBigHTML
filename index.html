The <base> element specifies a default URL and a default target (the "way" the link will open) for all links on a page. It is placed within the <head> element. It's like setting a custom home directory: ~/, an absolute address serving as a reference (absolute) to all other links on a page (relative). For languages that are written right-to-left, e.g. Hebrew or Arabic, HTML's got us covered with <bdi> (Bi-Directional Isolation) and <bdo> (Bi-Directional Override). The former will format Arabic or Hebrew text (that is written right-to-left) to comply with the other, left-to-right languages, so that HTML does not render it all broken and stuff. And vice versa too, maybe there's an Arabic article containing the name John Doe, which is English, which is written left-to-right. As a result, so as not to experience undefined behaviour, we would wrap John Doe in <bdi> tags. It would then show up very nicely on the page.
